## Creating 연산자

Creating 연산자는 새로운 `옵저버블`을 생성하는 데 사용되는 연산자입니다. 이러한 연산자들은 단 하나의 요소뿐만 아니라 배열, 시퀀스, 범위 등을 방출하도록 만들거나, 특정 조건에 따라 `옵저버블`의 동작을 제어하는 데 사용될 수 있습니다. 

### just

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| just(**element:**) | 이 연산자는 단 하나의 요소를 이벤트로 방출하는 `옵저버블`을 생성합니다. <br> 첫 번째 매개변수로 방출하고자 하는 요소를 전달해야 합니다. | - |

 ```swift
Observable<[String]>.just([✈️, ⭐️, ⚽️])
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

### of

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| of(**elements:**) | 이 연산자는 시퀀스의 각 요소를 하나씩 순차적으로 이벤트로 방출하는 `옵저버블`을 생성합니다. <br> 첫 번째 매개변수로 방출하고자 하는 요소를 차례로 전달해야 합니다. | - |

 ```swift
Observable<Int>.of(1, 2, 3, 4, 5)
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

### from

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| from(**array:**) | 이 연산자는 배열의 각 요소를 하나씩 순차적으로 이벤트로 방출하는 `옵저버블`을 생성합니다. <br> 첫 번째 매개변수로 방출하고자 하는 요소를 배열로 전달해야 합니다. | - |

 ```swift
Observable<Int>.from([1, 2, 3, 4, 5])
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

### range

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| range(**start:count:**) | 이 연산자는 시작값부터 일정 횟수 만큼 1씩 증가한 시퀀스를 생성하고, <br> 각 요소를 하나씩 순차적으로 이벤트로 방출하는 `옵저버블`을 생성합니다.  <br> start 매개변수에 시퀀스 시작값을 전달해야 합니다. <br> count 매개변수에 시퀀스 증가 횟수를 전달해야 합니다.  | - |

 ```swift
Observable<Int>.range(start: 0, count: 10)
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

`range` 연산자는 FixedWidthInteger 프로토콜을 준수하는 요소로만 시퀀스를 생성할 수 있습니다. 이런 이유로 `Double`이나 `CGFloat`을 타입으로 하는 시퀀스 생성은 불가능하며, 오직 `Int`나 `UInt` 타입으로 하는 시퀀스만 생성이 가능합니다.

### generate

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| generate(**initialState:condition:iterate:**) | 이 연산자는 시작값부터 정해진 규칙에 따라 시퀀스를 생성하고, <br> 각 요소를 하나씩 순차적으로 이벤트로 방출하는 `옵저버블`을 생성합니다. <br> intialState 매개변수에 시작값을 전달해야 합니다. <br> condition 매개변수에 시퀀스 생성을 중단할 조건을 검사하는 클로저를 전달해야 합니다. <br> 반환값이 false라면 시퀀스 생성을 중단합니다. <br> iterate 매개변수에 다음 시퀀스 요소를 만들어 반환할 클로저를 전달해야 합니다.  | - |

 ```swift
let airplane = "✈️"
let luggage = "🧳"

Observable<String>.generate(
    initialState: airplane, 
    condition: {
    $0.count < 10
}, iterate: {
    $0.count%2 == 0 ? $0 + airplane : $0 + luggage
})
.subscribe { print($0) }
.disposed(by: disposeBag)
 ```

위 예제 코드에서는 시퀀스 생성을 중단할 조건으로 문자열의 길이가 10 이상인 경우로 정하고 있고, 문자열의 길이가 2의 배수라면 ✈️을, 그게 아니라면 🧳을 현재값 뒤에 붙이고 있습니다. `generate` 연산자는 `range` 연산자와는 다르게, `FixedWidthInteger` 프로토콜을 준수하는 타입뿐만 아니라 문자열과 같은 다양한 타입에 사용될 수 있습니다. 

### repeatElement

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| from(**array:**) | 이 연산자는 단 하나의 요소를 무한정 이벤트로 방출하는 `옵저버블`을 생성합니다. <br> 첫 번째 매개변수로 방출하고자 하는 요소를 전달해야 합니다. | - |

 ```swift
Observable<Double>.repeatElement(123.0)
    .take(10)
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

`repeatElement` 연산자 사용 시 주의할 점은 이벤트를 **무한정** 방출한다는 점입니다. 이런 이유로 `take` 연산자로 방출되는 이벤트의 수를 적절히 제한해야 할 필요가 있습니다.

### deffered

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| deferred(**observableFactory:**) | 이 연산자는 조건에 따라 서로 다른 `옵저버블`을 생성합니다. <br> 첫 번째 매개변수로 `옵저버블`을 반환하는 클로저를 전달해야 합니다.| - |

 ```swift
let airbus = ["A220", "A330", "A404", "A606"]
let boeinig = ["737", "747", "787"]

var isAirbus = true
let observable = Observable<String>.deferred {
    return isAirbus ? Observable<String>.from(airbus) : Observable<String>.from(boeinig)
}

observable
    .subscribe { print($0); isAirbus = false }
    .disposed(by: disposeBag)

observable
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

위 예제 코드에서는 `isAirbus`가 참(True)이라면 에어버스 모델 명이 담긴 `옵저버블`을 반환하고, 그렇지 않다면 보잉 모델 명이 담긴 `옵저버블`을 반환합니다. 

### create

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| create(<**subscribe:**) | 이 연산자를 이용하면 `옵저버블`이 방출하는 이벤트를 직접 정의할 수 있습니다. <br> 첫 번째 매개변수로 (AnyObserver) -> Disposable 타입의 클로저를 전달해야 합니다. | - |

 ```swift
Observable<Int>.create { observer -> Disposable in
    for i in 1...5 {
        observer.onNext(i)
    }
    observer.onCompleted()
    
    return Disposables.create()
}
.subscribe { print($0) }
.disposed(by: disposeBag)
 ```

`옵저버블`이 구독자에게 이벤트를 방출할 때는 `onNext(element:)` 메서드를 호출해야 합니다. 모든 이벤트를 방출했으면 `onCompleted()` 메서드를 호출해 구독자에게 모든 이벤트 전달이 완료되었음을 알립니다. 마지막으로, Disposables.create()를 호출하여 Disposable을 생성하고, 이를 통해 클로저를 종료하고 구독의 해지 및 리소스 정리를 수행할 수 있습니다.

### empty

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| empty() | 이 연산자는 어느 이벤트도 방출하지 않는 빈 `옵저버블`을 생성합니다. | - |

 ```swift
Observable<Int>.from([1, 2, 3, 4, 5])
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```

### error

| 메서드               | 설명                 | 비고 |
|:-----------------: | :------------------ | :-- |
| error(<**error:**>) | 이 연산자는 에러 이벤트를 방출하는 `옵저버블`을 생성합니다. | - |

 ```swift
enum MyError: Error {
    case error
}

Observable<Void>.error(MyError.error)
    .subscribe { print($0) }
    .disposed(by: disposeBag)
 ```
