## What is RxSwift?

> **RxSwift** is a library for composing asynchronous and event-based code by using observable sequences and functional style operators, allowing for parameterized execution via schedulers. RxSwift in its essence simplifies developing asynchronous programs by allowing code to react to new data and process it in a sequential and isolated manner.

처음 반응형 프로그래밍(Reactive Programming)의 숨은 개념을 이해하는 건 무척이나 두려운 일처럼 보이기 합니다. 다만, 반응형 프로그래밍과 RxSwift에 익숙해지기 시작하면, 더 좋은 코드를 작성하는 데 많은 도움이 될 것입니다. 특히 비동기(Asynchronous)와 이벤트-기반 코드를 작성할 때, RxSwift는 적은 노력으로 코드 베이스(Code Base)를 유지보수하는 데 도움을 줍니다.

## Introduction to asynchronous programming

비동기 프로그래밍은 작업의 한 단위를 매인 애플리케이션 쓰레드(Main Application Thread)와 분리해 실행하며, 완료, 실패와 진행 상황을 호출 쓰레드(Calling Thread)에게 알려주는 병렬 프로그래밍(Parallel Programming)을 의미합니다. 아마 독자 여러분은 비동기 프로그래밍을 언제 사용하고, 어떤 이점과 문제점이 있는지 궁금할 수 있습니다.

주요 이점 중 하나는 비동기 프로그래밍을 사용함으로써 애플리케이션의 성능과 반응성 향상을 가져올 수 있습니다. 비동기 패턴에 특히 적합한 애플리케이션 중 하나는 계산을 하거나, 자원을 많이 소모하는 연산을 수행하는 동안 클라이언트 애플리케이션이 반응성 있는 UI를 제공해야 할 때입니다.

비동기 프로그래밍을 간단하게 설명드리기 위해, iOS 앱에서 몇 가지 예시를 들어보겠습니다. iOS 앱에서 우리는 아래와 같은 동작을 할 수 있습니다.

* 버튼 탭에 반응하기

* `UIView`에 트랜지션 애니메이션하기

* 서버로부터 작은 데이터를 다운로드하기

* 오디오나 비디오 재생하기

* 작은 데이터를 디스크에 저장하기

이러한 모든 작업은 동일한 시간에 일어나는 것처럼 보입니다. 뷰가 애니메이션되는 동안 앱의 오디오는 애니메이션이 끝날 때까지 일시 중지되지 않습니다. 프로그램의 모든 각기 다른 부분은 서로의 실행에 방해하지 않습니다. 서로 다른 실행 컨텍스트(Context)와 기기의 CPU 코어를 거쳐 서로 다른 쓰레드에서 다양한 종류의 작업을 수행할 수 있는 다양한 종류의 API가 있습니다. 

서로 다른 코드가 동일한 데이터 조각에 접근해 작업을 해야 할 때, 병렬로 실행되는 코드를 작성하는 건 무척 어려운 일입니다. 어느 코드가 먼저 데이터를 업데이트하는지, 최신 데이터를 읽어야 하는지 알기란 어렵기 때문입니다.

## Cocoa and UIKit asynchronous APIs

Apple은 개발자가 비동기 코드를 작성하는 데 도움을 주기 위해 iOS SDK에 매우 많은 API를 제공해오고 있습니다. 비동기 코드를 작성하기 위한 방법은 수 년 동안 많은 변화를 거쳐왔습니다.

아마 독자 여러분은 아래 방법을 주로 사용해 왔을 테고, 모바일 앱을 작성하는 데 매우 기본적이기에 다른 방법은 고려하지 않았을 겁니다.

* `NotificationCenter`: 코드가 언제든지 기기가 회전하거나, 소프트웨어 키보드가 화면에 나타나거나 사라지는 등 이벤트를 처리해줄 수 있습니다.

* The delegate pattern: 다른 객체를 대신하거나 협력하여 동작하는 다른 객체를 정의할 수 있게 해주는 패턴입니다.

* Grand Central Dispatch: 작업 조각의 실행을 추상화하는 데 도움을 줍니다. 코드 블록을 순차적이거나 동시에 실행하거나, 일정 시간 지연을 줄 수 있습니다.

* Closures: 코드에서 전달하고 최종적으로 사용할 수 있는 독립된 코드를 생성합니다.

## Foundation Of RxSwift

반응형 프로그래밍은 꽤 오랫 동안 존재해 왔지만, 그 핵심 개념들은 지난 몇 년 동안에 더욱 두드러지게 드러나게 되었습니다. 시간이 지날수록 앱이 더 복잡해지고 복잡한 비동기 UI를 관리해야 하는 필요성이 높아지고 있습니다. 서버 사이드(Server Side)에서는 반응형 시스템이 필수적으로 요구되고 있습니다.

Microsoft는 비동기 문제를 해결하고 확장이 용이한 개발을 위해 Reactive Extensions for .NET (Rx)라 불리는 새로운 클라이언트와 서버 사이드 프레임워크를 제공하고 있습니다. .NET용 Rx는 2012년부터 오픈소스로 제공되어 다른 언어와 플랫폼에서 동일한 기능을 다시 구현할 수 있게 허용되었습니다. 이로 인해 Rx는 크로스-플랫폼 표준으로 발전되었습니다.

오늘날에는 RxJs, RxKotlin, Rx.NET, RxScale, RxSwift 등이 있습니다. 모두 Reactive Extensions 설계를 기반으로 한 동일한 동작과 유사한 API로 구현되어 있습니다. 

RxSwift는 오리지널 Rx와 같은 모든 개념과 함께 작동합니다. 변하기 쉬운 상태를 다루고, 이벤트 스퀀스(Event Sequence)를 구성하는 걸 허용하며, 코드 분리, 재사용성과 디커플링(Decoupling)과 같은 아키텍처를 향상시킵니다.
